# Project Arc — Workflow de Aprendizado e Arquivamento de Conhecimento

> Um sistema pessoal para capturar, organizar, revisar e compartilhar conhecimentos adquiridos ao longo do tempo.

---

# Visão Geral

Objetivo:

* Aprender melhor
* Construir um acervo pessoal de conhecimento
* Praticar escrita
* Praticar inglês
* Produzir conteúdo quando fizer sentido
* Evitar reaprender a mesma coisa várias vezes

Fluxo:

```text
Aprendi algo
     ↓
Crio nota
     ↓
Transformo em artigo
     ↓
Traduzo para inglês
     ↓
Gravo vídeo (opcional)
     ↓
Publico
```

---

# Etapa 1 — Captura Rápida

Sempre que aprender algo relevante:

* Swift
* Flutter
* IA
* Investimentos
* Música
* Livros
* Produtividade
* Ferramentas
* Configurações
* Processos

Criar uma nota simples.

Exemplo:

```markdown
# Swift 6 - Sendable

## O que é?

...

## O problema que resolve

...

## Exemplo

...

## Minha conclusão

...
```

Objetivo:

* Registrar enquanto o conhecimento está fresco.
* Não buscar perfeição.
* Apenas capturar.

---

# Etapa 2 — Transformar em Artigo

Quando a nota estiver madura:

```text
Nota → Artigo
```

Estrutura sugerida:

```markdown
# O que aprendi sobre Sendable no Swift 6

## Contexto

## O problema

## Como funciona

## Exemplo simples

## Quando usar

## Conclusão
```

Objetivo:

Explicar para o "Julio do futuro".

Se o Julio daqui a dois anos entender rapidamente o artigo, ele foi bem escrito.

---

# Etapa 3 — Tradução para Inglês

Após finalizar o artigo em português:

```text
Artigo PT-BR → Artigo EN
```

Benefícios:

* Vocabulário técnico
* Escrita em inglês
* Revisão dos conceitos
* Preparação para entrevistas e discussões técnicas

---

# Etapa 4 — Vídeo (Opcional)

Somente para conteúdos realmente importantes.

Fluxo:

```text
Artigo → Roteiro → Vídeo
```

O vídeo pode ser simples:

* Sem edição complexa
* Sem preocupação com audiência
* Sem preocupação com visual

Objetivo:

* Praticar speaking
* Melhorar pronúncia
* Fixar o conhecimento

Quando você consegue ensinar um assunto, normalmente aprendeu de verdade.

---

# Estrutura do Repositório

```text
Knowledge
│
├── Notes
│
├── Blog Posts (PT-BR)
│
├── Blog Posts (EN)
│
└── Videos
```

---

# O Erro Que Quero Evitar

Não começar pelo vídeo.

Fluxo ruim:

```text
Aprendi algo
↓
Vou gravar um vídeo
↓
Nunca gravo
```

Fluxo recomendado:

```text
Aprendi algo
↓
Crio uma nota
↓
Transformo em artigo
↓
Traduzo para inglês
↓
Gravo vídeo (se fizer sentido)
```

# Nome do Projeto

## Project Arc

Inspirado no Arc Reactor.

Representa:

* Energia para novos projetos
* Engenharia
* Experimentação
* Aprendizado contínuo

---

# Convenção dos Artigos

Formato:

```text
Project Arc #001 - Título
```

Exemplos:

```text
Project Arc #001 - Configurando Jekyll no GitHub Pages

Project Arc #002 - Appium + Python para iOS

Project Arc #003 - Como analiso relatórios de FIIs

Project Arc #004 - Setup local de IA com LM Studio

Project Arc #005 - Migrando um módulo para Swift 6

Project Arc #006 - Introdução ao Flutter BLoC

Project Arc #007 - Como organizo meus testes E2E

Project Arc #008 - Configurando um agente local
```

A regra principal é simples:

> Não quero aprender a mesma coisa duas vezes.

Cada Project Arc deve funcionar como uma cápsula de conhecimento deixada para o meu eu do futuro.
